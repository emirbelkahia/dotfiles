# Changelog

All notable changes to this repository are documented in this file.

## 2026-03-04 (4)

### Changed

- `bin/sms-export` — HTML viewer and UX improvements:
  - **Search in message content** — search bar now searches both conversation names and message content. Sidebar preview updates to show a snippet of the first matching message when the match is in content (not the name). When opening a matched conversation, all matching bubbles are highlighted in yellow and auto-scroll to the first match.
  - **Visual progress bars** — replaced plain counters with `█░` progress bars in both the message loading phase and the attachment export phase.
  - **Date range header** — sidebar now shows the full temporal scope of the backup (`📅 Mar 15, 2022 → Mar 4, 2026`) so you know exactly which period of messages is covered, even when opening an old export months later.
  - **sms-export attribution** — "Generated by sms-export" line now links to `github.com/emirbelkahia/dotfiles` with a tooltip (`sms-export — custom CLI tool by Emir Belkahia`). Makes it easy to trace which version of the tool produced a given export.
- `README.md`, `--help` — updated documentation to reflect all current features and prerequisites.

## 2026-03-04 (3)

### Changed

- `bin/sms-export` — major iteration after first working run. Full list of changes since initial add:
  - **Encrypted backup support** via `iphone-backup-decrypt` (jsharkey13). Detects `IsEncrypted` flag in `Manifest.plist`, prompts for password, decrypts on the fly. No unencrypted backup required.
  - **Dedicated venv** at `~/.venv/sms-export/` — `iphone-backup-decrypt` installed there by `install.sh`, auto-activated at script startup via `os.execv()` + `sys.prefix` comparison. No manual activation, no global pip install, no dotfiles philosophy violation.
  - **File-based attachments** — photos, videos, audio and other files exported to a sibling folder `~/Documents/YYYY-Month-DD - Backup iMessage iPhone/` and referenced via relative HTML paths. No base64 embedding.
  - **Contact name resolution** — reads `AddressBook.sqlitedb` from the backup (encrypted or not). Phone numbers matched on last 9 digits to handle +33 vs 06 prefix differences. Emails matched case-insensitively. Falls back to raw phone/email if not found.
  - **Progress counter** — prints `N/total messages...` with `\r` overwrite during message loading.
  - **Fixed attachment hash domain** — `~/Library/SMS/Attachments/` paths belong to `HomeDomain`, not `MediaDomain`. Bug caused all images/videos from iMessage to show as "not found in backup". Now correctly computes `HomeDomain-Library/SMS/Attachments/…` hash.
  - **Plugin payload filtering** — `.pluginPayloadAttachment` files (stickers, link previews, Apple Pay, iMessage app extensions) are silently skipped. They are iOS-internal binary blobs with no displayable content.
  - **Multi-backup selection** — reads `Info.plist` for device name and last backup date, displays human-readable list instead of UUIDs.
  - **Full Disk Access error handling** — wraps `BACKUP_DIR.iterdir()` in `try/except PermissionError` with clear guidance (System Settings → Privacy & Security → Full Disk Access).
  - All UI messages in English.
- `install.sh` — added `~/.venv/sms-export/` venv creation and `iphone-backup-decrypt` install in the macOS section.

## 2026-03-04 (2)

### Added

- `bin/sms-export` — exports iPhone messages from a local Finder backup to HTML. Auto-detects the SMS SQLite database, reads all iMessage/SMS conversations and attachments. Output: `~/Documents/YYYY-Month-DD - Backup iMessage iPhone.html` + a sibling `~/Documents/YYYY-Month-DD - Backup iMessage iPhone/` folder containing all attachment files (photos, videos, audio, other). HTML references attachments via relative paths — no base64 embedding. Images displayed inline via `<img>`, video/audio playable via native HTML5 tags, other files as download links. If several backups exist, lists them by device name and date (read from `Info.plist`) and prompts to choose. Colored `--help`. No iCloud, no paid tools, no pip dependencies.

## 2026-03-03

### Added

- `bin/ooo` — Gmail out-of-office setter in pure bash (curl + jq + nc). No Python, no venv. Supports interactive mode, direct args (`ooo 2026-03-10 2026-03-17`), `--setup` for first-time OAuth via localhost callback, and `--help`. Config lives in `~/.config/ooo/` (never committed — `credentials.json` in 1Password).
- `bin/` convention — personal CLI scripts versioned in `dotfiles/bin/`, symlinked to `~/bin` on PATH. Add scripts here to make them available on any machine after `./install.sh`.
- `jq` added to Homebrew install in `install.sh`.
- `$HOME/bin` added to PATH in `.zshrc`.
- `ln -sf "$HOME/dotfiles/bin" "$HOME/bin"` added to `install.sh` symlink section.

## 2026-03-02

### Added

- `zsh-autosuggestions` — fish-like command suggestions based on history. Installed via Homebrew, sourced in `.zshrc`.
- Finder path bar (`ShowPathbar`) — shows the full file path at the bottom of every Finder window permanently, without holding Option.
- `hammerspoon/init.lua` — time overlay triggered by `Cmd+Alt+T`. Displays current time in large pink text on a dark semi-transparent background for 2 seconds. Installed via `brew install --cask hammerspoon`, symlinked by `install.sh`.

## 2026-03-01

### Added

- `macos.sh` — macOS `defaults write` speed optimizations script with full revert support. Disables animations (window open/close, Dock launch bounce, Mission Control, Quick Look, Finder), sets keyboard repeat to max speed, configures Finder (search current folder, full path in title, no .DS_Store on network/USB), screenshots to ~/Downloads without shadow/thumbnail, save/print dialogs expanded by default. Run `./macos.sh revert` to undo everything.
